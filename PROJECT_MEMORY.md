## 当前有效约束
- 商品主数据已从“品牌”切换为“供应商”，涉及商品类型、筛选、表单、基础资料时优先使用 `supplier` 语义。
- 销售角色前端权限已收口：
  - 销售仅保留“数据概览 / 商品管理 / 订单管理”
  - 销售端不显示供应商信息、规格成本价、商品导入/新建/编辑/删除/调库存入口
- 前端已做金额字段归一化，处理后端把数字返回为字符串的情况；若再出现 `toFixed` 类报错，先检查接口字段类型，再补归一化。
- 前端 `.env` 文件当前按团队约定允许提交，但默认仍不要随意改动环境配置。

## 当前功能状态
- 订单创建：
  - 新建订单主流程使用 `soldPrice` 作为录单价格口径。
  - `src/pages/Orders/OrderForm.tsx` 支持 web 端扫码录单，PAD 可用扫码枪输入标签码后直接回车带入规格。
  - 扫码命中已存在规格时会自动累加数量，并受可售库存上限约束。
  - 选择规格弹窗现在会展示商品主图缩略图；无图时显示“无图”占位，方便区分商品。
  - 订单列表的“下单时间”列已支持升序/降序切换，排序参数会传到后端分页查询，不再只在当前页本地排序。
- 商品标签：
  - 商品列表和商品详情支持打印标签。
  - 标签当前展示二维码、人眼编码、商品名、款号、颜色、尺码、销售价、主图。
  - 标签导出已针对 iPad Safari 做过兼容处理，避免二维码缺失。
  - 标签弹窗现已同时支持“批量下载全部标签”和“单独下载当前标签”。
  - 批量下载已从“浏览器逐张触发下载”改为“前端打 ZIP 后单文件下载”，以规避 iPad 上只下载第一张的问题。
- 商品导入：
  - 商品列表已支持批量导入，路由为 `/products/import`。
  - 支持 Excel、本地截图识别，以及“Pad 导入 Excel”服务端直传解析三种导入来源。
  - 前端保留同一商品内颜色/尺码组合去重等必要校验，但不再把“款号重复”作为前端拦截错误。
- 统计与客户：
  - 统计页已重构为“销售统计 / 成本统计”双 Tab。
  - 客户年龄段已接入订单创建表单。
  - 客户档案当前作为统计页内的只读展示入口，不再放在系统配置页。

## 部署与运行经验
- 前端生产目录使用标准 git 工作区。
- 如需在生产机构建前端，曾确认需要显式切到 `/usr/local/node20-bin` 下的 Node 20；若首页再次出现 `500`，优先检查 `dist` 是否存在以及构建时 Node 版本是否正确。
- 历史上发布曾被生产机 `package-lock.json` 漂移拦截；当前方案是前端使用 `npm ci`，若再遇到发布前工作区脏的问题，先检查锁文件和本地 stash。

## 已知风险与待关注项
- `vite build` 仍有大体积 chunk warning，主要集中在 `vendor-antd`，当前不阻塞功能开发，但后续若继续扩展页面需留意包体积。
- 后台 web 端扫码录单当前复用 `/api/mobile/products/by-code`；若后续后台和移动端语义继续分化，建议抽成通用商品查询能力。
- 标签下载当前仍是浏览器侧逐张触发；若后续 iPad 出现多文件下载拦截，可考虑改成 ZIP 打包或后端生成。
