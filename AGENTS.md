# AGENTS 工作规范

## 作用范围
- 本文件对整个仓库生效。

## 环境基线
- 本仓库默认使用 `npm` 管理依赖和脚本。
- 默认本地地址为 `http://127.0.0.1:5173/`。
- 前端默认通过 `/api` 代理连接真实后端。
- 除非用户明确要求，不启用 mock 模式。

## 常用命令
- 安装依赖：`npm install`
- 启动开发：`npm run dev -- --host 127.0.0.1 --port 5173`
- 构建：`npm run build`

## 必要会话流程
1. 每次新会话开始时，先读取 `PROJECT_MEMORY.md`。
2. 每次会话结束前，更新 `PROJECT_MEMORY.md`。
3. 收尾回复固定包含三项：
   - 改了什么
   - 如何验证
   - 剩余风险或待办

## 变更边界
- 仅修改与当前用户需求直接相关的文件。
- 未经用户要求，不做顺手重构。
- 如果发现与当前任务冲突的意外本地改动，先暂停并确认再继续。

## 使用方式提醒
- 如果发现用户当前使用方式低效、容易误操作，或与仓库现有工作流不一致，应直接指出。
- 给出更优用法时，优先提供当前项目可直接执行的命令、路径或调用方式，不只给抽象建议。
- 如果用户的做法能完成任务但成本更高，也应主动说明更省事、更稳定或更可复用的替代方案。
- 涉及部署、巡检、构建、环境配置时，优先提醒可复用的脚本、skill 或标准入口，避免重复手工操作。

## 验证规则
- 代码修改后，至少执行一项相关验证（`build`、测试或定向运行验证）。
- 如无法验证，必须明确说明未验证项。

## 安全规则
- 回复中不得输出密钥、密码、令牌等敏感信息。
- 未经明确要求，不修改 `.env` 文件。

## Git 规则
- 提交信息清晰可读，优先使用 Conventional Commit 风格。
- 未经明确要求，不执行 `commit --amend`。
- 未经明确要求，不执行强推（force push）。

## 记忆更新规则
- 内容保持简洁、事实化。
- 仅记录高价值信息：环境或配置变化、运行与验证结果、问题与根因、关键决策和约束。
- 更新 `PROJECT_MEMORY.md` 时使用固定模板：会话日期、变更内容、验证结果、遗留问题或风险。
- 避免重复陈旧或低价值信息。
